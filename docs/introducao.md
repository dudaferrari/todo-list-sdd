# Introdução

Este projeto é uma aplicação de lista de tarefas desenvolvida com um monorepo que separa frontend e backend em módulos distintos.

O objetivo principal é demonstrar uma implementação simples, organizada e alinhada à constituição do projeto. A solução prioriza a arquitetura MVC, a separação de responsabilidades, a facilidade de uso e a manutenção do código, mantendo os dados em memória para o escopo atual.

## Princípios adotados

- Arquitetura MVC obrigatória.
- Frontend em React e backend em Express com JavaScript.
- Estrutura de monorepo para organizar as camadas do sistema.
- Armazenamento em memória, sem dependência de banco de dados.
- Testes para garantir a qualidade das funcionalidades principais.
- Documentação clara para facilitar a compreensão e evolução do projeto.
